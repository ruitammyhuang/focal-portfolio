June 2026
Portfolio
v0.1 — Portfolio Launched
System state

Focal v2 modular architecture is stable and in daily use. Core features operational: sync engine, Quick Add, Focus Tasks view, Workload Dashboard. Five projects actively tracked: EDG 6648 Summer Instruction, EDG 6648 Course Design, Beyond LXD Label Scoping Review, CS Ed EdD Cohort 2 Workshop, Program Management.

What happened this cycle

Portfolio created as a structured artifact for documenting the human–AI collaboration process. Initial content covers all four portfolio goals: knowledge frameworks, competency analysis, design artifacts, and lessons learned. This is the first time the development process itself has been treated as a research site.

New insight

Building the portfolio forced articulation of the collaboration process in a way that revealed patterns not previously named. Most notably: "understanding lag" as a new failure mode specific to AI-augmented development, and "context design" as an underappreciated meta-competency.

What's open / next

Portfolio needs: design artifacts page (linking to existing docs), deeper competency analysis as the case continues, and first social media sharing test. System needs: refinements to Workload Dashboard work type categorization.

May 2026
System
Focal v2 — Modular Architecture Refactor
What changed

Monolithic notion_wbs_sync.py (~3,100 lines) refactored into a proper Python package: focal/app.py, config.py, notion_client.py, sync_engine.py, tasks.py, with route blueprints and HTML templates. Entry point moved to main.py. _MANIFEST.md Tier System introduced for context management.

Why it was done

Monolith became too large for efficient AI collaboration — locating the right section required too much context. The refactor reduced per-session cognitive load and made the codebase easier to reason about for both human and AI.

Key lesson

The refactor wasn't triggered by a feature failure — the system worked. It was triggered by collaboration friction. This is an important data point: in human–AI projects, architecture quality directly affects collaboration quality, not just system quality.

Early 2026
System
Focal v1 — First Automation (Monolithic Script)
What was built

Single Python/Flask script that syncs per-project Notion WBS databases into a Master WBS Tasks database. Flask UI added for sync triggering and Quick Add. Work Sessions auto-creation added. Core sync rules established: one-directional, idempotent, status only set on CREATE.

Key design decision

One-directional sync (Project WBS → Master only). This was not obvious initially — the first implementation was bidirectional and immediately caused a status overwrite bug. The fix revealed a core architectural principle.
